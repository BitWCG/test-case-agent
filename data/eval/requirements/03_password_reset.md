# 密码重置功能需求文档

## 功能概述
用户忘记密码时，可通过注册邮箱重置密码。系统发送重置链接到邮箱，用户点击链接后设置新密码。

## 详细需求

### 1. 重置流程
- 用户点击"忘记密码"，输入注册邮箱
- 系统校验邮箱是否已注册
- 如果已注册，发送重置密码邮件（含重置链接）
- 无论邮箱是否已注册，统一提示"如果该邮箱已注册，将收到重置邮件"
- 用户点击邮件中的重置链接，进入重置密码页面
- 输入新密码和确认密码，提交后完成重置

### 2. 重置链接
- 重置链接有效期 30 分钟
- 过期后需重新申请
- 每个重置链接只能使用一次
- 同一邮箱同时只能有一个有效的重置链接（新链接生成时旧链接失效）

### 3. 新密码规则
- 不能与最近 3 次使用过的密码相同
- 其余规则与注册密码规则一致（8-20 位，至少 3 种字符类型）

### 4. 安全要求
- 重置链接发送限流：同一邮箱 1 小时内最多 3 次
- 重置链接发送限流：同一 IP 1 小时内最多 5 次
- 重置成功后，所有已登录的设备自动登出
- 重置成功后，发送邮件通知用户"密码已被重置"

### 5. 异常处理
- 如果重置链接已过期，提示用户重新申请
- 如果重置链接已使用，提示用户重新申请
- 如果 token 无效，统一返回"链接无效或已过期"

## 接口定义

### POST /api/v1/password/forgot

**请求体：**
```json
{
  "email": "string"
}
```

**响应（200）：**
```json
{
  "code": 0,
  "message": "如果该邮箱已注册，将收到重置邮件"
}
```

### POST /api/v1/password/reset

**请求体：**
```json
{
  "token": "string",
  "new_password": "string",
  "confirm_password": "string"
}
```

**成功响应（200）：**
```json
{
  "code": 0,
  "message": "密码重置成功"
}
```

**失败响应 — Token 无效/过期（400）：**
```json
{
  "code": 400,
  "message": "链接无效或已过期"
}
```

**失败响应 — 密码不符合规则（400）：**
```json
{
  "code": 400,
  "message": "新密码不符合安全要求",
  "details": "密码长度需为 8-20 位，且包含至少 3 种字符类型"
}
```

**失败响应 — 密码与历史重复（400）：**
```json
{
  "code": 400,
  "message": "新密码不能与最近使用过的密码相同"
}
```

### POST /api/v1/password/verify-token

**请求体：**
```json
{
  "token": "string"
}
```

**成功响应（200）：**
```json
{
  "code": 0,
  "message": "链接有效",
  "data": {
    "email": "user@example.com",
    "expires_in": 600
  }
}
```

**失败响应（400）：**
```json
{
  "code": 400,
  "message": "链接无效或已过期"
}
```
