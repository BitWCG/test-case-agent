# 用户注册功能需求文档

## 功能概述
新用户通过注册页面创建账号，填写必要信息后完成注册，系统自动发送验证邮件。

## 详细需求

### 1. 注册流程
- 用户访问注册页面，填写注册表单
- 表单包含：用户名、邮箱、密码、确认密码、验证码
- 提交后系统校验所有字段
- 校验通过后发送验证邮件到注册邮箱
- 用户点击邮件中的链接激活账号
- 激活后可以使用账号登录

### 2. 用户名规则
- 长度：4-20 个字符
- 允许：字母、数字、下划线、中文
- 不允许：纯数字、空格、特殊符号
- 用户名唯一，不可重复

### 3. 邮箱规则
- 必须为合法的邮箱格式
- 邮箱唯一，一个邮箱只能注册一个账号
- 支持常见邮箱域名（QQ、163、Gmail、Outlook 等）

### 4. 密码规则
- 长度：8-20 位
- 必须包含：大写字母、小写字母、数字、特殊字符中至少 3 种
- 不允许：连续 3 位相同字符、与用户名相同
- 密码强度分三级：弱（8-10 位）、中（11-15 位）、强（16-20 位）

### 5. 验证码
- 4 位数字验证码
- 有效期 5 分钟
- 输入错误 3 次后需重新获取验证码
- 同一邮箱 1 小时内最多发送 5 次验证码

### 6. 激活链接
- 激活链接有效期 24 小时
- 过期后需重新发送验证邮件
- 激活链接包含一次性 token，使用后失效

### 7. 安全要求
- 密码传输使用 HTTPS 加密
- 密码存储使用 bcrypt 哈希
- 注册接口限流：同一 IP 每分钟最多 3 次请求
- 验证码发送限流：同一邮箱每小时最多 5 次

## 接口定义

### POST /api/v1/register

**请求体：**
```json
{
  "username": "string",
  "email": "string",
  "password": "string",
  "confirm_password": "string",
  "captcha": "string"
}
```

**成功响应（201）：**
```json
{
  "code": 0,
  "message": "注册成功，请查收验证邮件",
  "data": {
    "user_id": 1,
    "activation_expires_in": 86400
  }
}
```

**失败响应 — 用户名已存在（409）：**
```json
{
  "code": 409,
  "message": "用户名已被注册",
  "field": "username"
}
```

**失败响应 — 邮箱已注册（409）：**
```json
{
  "code": 409,
  "message": "该邮箱已注册",
  "field": "email"
}
```

**失败响应 — 验证码错误（400）：**
```json
{
  "code": 400,
  "message": "验证码错误",
  "field": "captcha",
  "remaining_attempts": 2
}
```

### POST /api/v1/activate

**请求体：**
```json
{
  "token": "string"
}
```

**成功响应（200）：**
```json
{
  "code": 0,
  "message": "账号激活成功"
}
```

**失败响应 — Token 过期（410）：**
```json
{
  "code": 410,
  "message": "激活链接已过期，请重新发送验证邮件"
}
```
