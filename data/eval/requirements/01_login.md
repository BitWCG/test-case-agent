# 用户登录功能需求文档

## 功能概述
用户通过用户名和密码登录系统，成功后获取访问令牌（token），失败则返回相应错误信息。

## 详细需求

### 1. 登录流程
- 用户在登录页面输入用户名和密码
- 系统验证用户名和密码的正确性
- 验证成功：返回 JWT token，token 有效期为 2 小时
- 验证失败：返回错误码 401 和错误信息

### 2. 用户名规则
- 长度：4-20 个字符
- 允许：字母、数字、下划线
- 不允许：空格、特殊字符、中文

### 3. 密码规则
- 长度：6-20 位
- 只允许：字母和数字
- 不区分大小写（存储时统一转为小写）

### 4. 账号锁定
- 连续 5 次登录失败后，锁定账号 30 分钟
- 锁定期间，即使输入正确密码也返回错误码 423
- 锁定期间返回提示信息："账号已锁定，请 30 分钟后再试"
- 30 分钟后自动解锁，失败次数清零

### 5. 记住登录状态
- 用户可勾选"记住我"选项
- 勾选后 token 有效期延长至 7 天
- 未勾选则使用默认 2 小时有效期

### 6. 安全要求
- 密码传输使用 HTTPS 加密
- 密码存储使用 bcrypt 哈希
- 登录接口限制：同一 IP 每分钟最多 10 次请求
- 超过限流返回错误码 429

## 接口定义

### POST /api/v1/login

**请求体：**
```json
{
  "username": "string",
  "password": "string",
  "remember_me": false
}
```

**成功响应（200）：**
```json
{
  "code": 0,
  "data": {
    "token": "eyJhbGci...",
    "expires_in": 7200,
    "user": {
      "id": 1,
      "username": "testuser"
    }
  }
}
```

**失败响应（401）：**
```json
{
  "code": 401,
  "message": "用户名或密码错误",
  "remaining_attempts": 3
}
```

**锁定响应（423）：**
```json
{
  "code": 423,
  "message": "账号已锁定，请 30 分钟后再试",
  "locked_until": "2026-07-01T12:30:00Z"
}
```

**限流响应（429）：**
```json
{
  "code": 429,
  "message": "请求过于频繁，请稍后再试",
  "retry_after": 60
}
```
